

# Исправление: показ модулей из всех разделов при копировании/перемещении

## Проблема

В диалоге копирования/перемещения (`CopyMoveDialog`) дерево модулей (`ModuleTreeSelector`) фильтрует по текущему `menu_section_key`. Модули «База знаний», «Закрой год» и «Бухгалтерия как бизнес» находятся в разных разделах, поэтому показывается только один из них.

Дополнительный баг: при смене раздела `targetModuleId` не сбрасывается на `null`, что может привести к некорректной операции.

## Решение

### Файл: `src/components/admin/trainings/CopyMoveDialog.tsx`

1. **Сброс `targetModuleId`** при смене `sectionKey`: добавить обёртку над `setSectionKey`, которая также вызывает `setTargetModuleId(null)`.

2. **Подсказка** под `ContentSectionSelector`: добавить текст "Смените раздел, чтобы увидеть модули из другого раздела" (`text-xs text-muted-foreground`), чтобы пользователю было очевидно, что дерево модулей зависит от выбранного раздела.

## Что НЕ трогаем

- `ModuleTreeSelector` (фильтрация по section — корректное поведение)
- `ContentSectionSelector`
- Edge function
- Остальные компоненты
